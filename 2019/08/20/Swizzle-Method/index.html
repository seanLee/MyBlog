Swizzle Method

发表于 2019-08-20

简介

作为一门动态编程语言,要决定一个Objectived-C方法的最终执行,不仅要考虑编译阶段,同时也要考虑运行阶段.因此Objectived-C提供runtime system机制来保证语言的动态性.

使用

在日常开发中,有关Runtime的使用有很多种场景,今天要介绍的只是其中一种,即动态交换方法.在此之前,需要先介绍几个相关的知识点:

Class

typedef struct objc_method *Method;

struct objc_method {
SEL _Nonnull method_name OBJC2_UNAVAILABLE;
char * _Nullable method_types OBJC2_UNAVAILABLE;
IMP _Nonnull method_imp OBJC2_UNAVAILABLE;
} OBJC2_UNAVAILABLE;

typedef id (*IMP)(id, SEL, …);

void Swizzle(Class c, SEL origSEL, SEL newSEL) {
Method origMethod = class_getInstanceMethod(c, origSEL);
Method newMethod = nil;
if (!origMethod) {
origMethod = class_getClassMethod(c, origSEL);
if (!origMethod) {
return;
}
newMethod = class_getClassMethod(c, newSEL);
if (!newMethod) {
return;
}
}else{
newMethod = class_getInstanceMethod(c, newSEL);
if (!newMethod) {
return;
}
}

if(class_addMethod(c, origSEL, method_getImplementation(newMethod), method_getTypeEncoding(newMethod))){
    class_replaceMethod(c, newSEL, method_getImplementation(origMethod), method_getTypeEncoding(origMethod));
}else{
    method_exchangeImplementations(origMethod, newMethod);
}

}

通过传进来的Class类获取实例方法,如果origMethod不为空就表示当前交换是针对类的实例方法,反之则分别获取

如果本身类里面不存在origSEL的实现(即实在父类中现该方法,子类中并没有重写),如果直接使用会导致子类中的

@implementation UIViewController (Swizzle)

  • (void)load {
    static dispatch_once_t onceToken;
    dispatch_once(&onceToken, ^{ 
    swizzleAllViewController();
    });
    }

void swizzleAllViewController() {
Swizzle([UIViewController class], @selector(viewWillAppear:), @selector(customviewWillAppear:));
}

  • (void)customviewWillAppear:(BOOL)animated {
    [self customviewWillAppear:animated];
    }
    `

    注意

    1.因为+(void)load会在类被加载到程序时被runtime主动调用一次,所以将交换的时机放在该函数内.
    2.一般情况下+(void)load方法只会调用一次,但不排除会主动被人调用,所以加上dispatch_once方法确保函数交换只会被执行一次.
